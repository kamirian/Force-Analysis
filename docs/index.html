Why Average Error Is Not Enough

Current FP evaluations overwhelmingly rely on a single number — the MAE or RMSE of force magnitude. This is convenient but incomplete.

The problem: A model can predict forces that point in completely wrong directions, or produce catastrophically large errors on a small fraction of atoms — and both failure modes are masked by the average.

These failures have real consequences. Static relaxations and NEB calculations require force convergence of 0.001–0.01 eV/Å. Misdirected or catastrophically large forces during MD can destabilize trajectories.

Metric Definitions

Two complementary per-atom metrics, evaluated across three physically motivated force regimes.

|ΔF|
Force magnitude error — eV/Å
|ΔF| = | |F_FP| − |F_DFT| |
How far off is the predicted force strength? Drives errors in bond lengths, lattice constants, and energy barriers. MAE is the primary metric; RMSE captures catastrophic outliers.
|Δθ|
Force direction error — degrees
|Δθ| = arccos( F_FP · F_DFT / (|F_FP| |F_DFT|) )
How wrong is the predicted force direction? Large angular error pushes atoms toward wrong energy minima even when magnitude looks acceptable. Only computed for |FDFT| > 0.01 eV/Å.

Fraction metrics

frac(<δ & <α) ↑
DFT-substitutable fraction
% atoms: |ΔF| < δ AND |Δθ| < α
Joint criterion requiring both low magnitude and accurate direction. Reported for (δ,α) = (0.01 eV/Å, 1°) and (0.01 eV/Å, 20°). The fraction of atoms where the FP force can substitute DFT.
frac(>1 eV/Å) ↓
Catastrophic error fraction
% atoms: |ΔF| > 1 eV/Å
Even a small fraction of atoms with |ΔF| > 1 eV/Å can crash an MD simulation. This metric is invisible to average error but highly consequential.

Three analysis regimes

DFT-substitutable

Highly accurate predictions

|ΔF| < 0.01–0.1 eV/Å and |Δθ| < 1–20°. Required for converged relaxations and NEB barriers. Jointly satisfying both is rare (<20% of atoms for most FPs).

Catastrophic failures

Large-error atoms

|ΔF| > 1 eV/Å. Rarely reported, but highly consequential — a small fraction of these can crash MD or corrupt a trajectory.

Far-from-equilibrium

High-force configurations

|FDFT| ≥ 1 eV/Å. About 26.9% of MatPES-PBE atoms fall here vs ~6% in MPtrj. Most benchmarks under-test this regime.

How to Submit Your Model

Evaluate your FP on these datasets and appear on the leaderboard using the same pipeline.

Reproducibility: All analysis code is open-source. Follow this protocol to get results directly comparable to the leaderboard.

Set up the environment

Clone the repo and install dependencies. Each FP has its own environment — see the README for per-model installation (ASE, MACE, CHGNet, MatGL, fairchem, etc.).

git clone https://github.com/kamirian/Matpes_analysis.git
pip install -r requirements.txt   # then install your FP's package

Download the benchmark data

Pull LFS files and download large PBE/r2SCAN JSONs (links in README). Place in data/.

Run inference with your model

Use the generator notebooks (generate/run_pbe.ipynb, etc.) which accept any ASE-compatible calculator. Output: per-atom arrays for each structure.

all_results["MyModel"] = {
    "all_deltaF":     [...],   # |F_FP - F_DFT| per atom (eV/Å)
    "all_deltaTheta": [...],   # angle(F_FP, F_DFT) per atom (degrees)
    "all_F_dft_mags": [...],   # |F_DFT| per atom (eV/Å)
}

Compute and report metrics

Run the analysis notebook with your model in all_results. Report the 7 leaderboard columns: MAE/RMSE |ΔF|, MAE/RMSE |Δθ|, frac(<0.01), frac(<0.01 & <1°/20°), frac(>1 eV/Å).

Open a GitHub issue

Submit results by opening an issue with your model name, architecture, training data, and computed metrics. We will review and add you to the leaderboard.
