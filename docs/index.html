Why Two Error Metrics?

Standard force benchmarks collapse everything into a single number — the root-mean-square error of the force vector. This hides an important distinction: a model can predict the right force magnitude while pointing in the completely wrong direction, or vice versa. Both failure modes cause different problems in practice (bad geometry vs. bad dynamics).

|ΔF|
Force magnitude error — eV/Å
How far off is the predicted force strength? Drives errors in bond lengths, lattice constants, and relaxation convergence.
|Δθ|
Angular error — degrees
How wrong is the predicted force direction? A large angular error drives atoms toward wrong minima even when the magnitude looks reasonable.

By analyzing both metrics — across force magnitude regimes (near-equilibrium vs. far-from-equilibrium), across models, and across functionals — this work gives a multi-dimensional view of MLIP quality.

Data format

Each result JSON has the structure:

{
  "model_name": {
    "all_F_dft_mags":   [...],  // DFT force magnitudes |F_DFT| in eV/Å
    "all_deltaF":       [...],  // Force magnitude errors |ΔF| in eV/Å
    "all_deltaTheta":   [...],  // Angular errors |Δθ| in degrees
    "original_indices": [...]   // Structure indices in the source dataset
  }
}
